### **3. 当前用户数据结构**

你需要提供当前用户 mock 或数据库字段，例如：

```ts
type User = {
  id: string
  name: string
  avatar?: string
  phone?: string
  email?: string
  company?: string
  title?: string
}
```

我重点需要知道：

```txt
当前用户表 / mock 里已有字段有哪些？
是否已经有头像？
是否已经有手机号？
是否已经有飞书 ID？
是否已经有微信绑定状态？
是否已有角色字段？
```

第一版即使是假数据，也要知道字段名，避免后面返工。

---

### **4. 积分规则**

这是最关键的业务规则。

你需要定一个第一版规则，可以先很简单：

```txt
阅读文章：+几分？
收藏文章：+几分？
发布复盘：+几分？
完成答题：+几分？
每日登录：+几分？
每日上限：是否需要？
重复阅读是否加分？
取消收藏是否扣分？
管理员是否能手动调整积分？
```

建议第一版先用：

```txt
每日登录 +5
阅读文章 +2
收藏文章 +1
发布复盘 +10
完成答题 +5
每日积分上限 50
```

## **P2：后面再补，不影响第一版界面**

### **8. 后台和数据库是否现在就接**

你需要明确第一版是：

```txt
A. 纯前端 mock 页面
B. 前端 mock + 预留 API 类型
C. 直接接数据库 / API
```

我的建议是：

```txt
第一版选 B：前端 mock + 预留 API 类型。
```

也就是先做：

```txt
src/app/me/page.tsx
src/data/me.mock.ts
src/types/me.ts
```

后续再接：

```txt
/api/me/overview
/api/me/points
/api/me/learning-events
```
