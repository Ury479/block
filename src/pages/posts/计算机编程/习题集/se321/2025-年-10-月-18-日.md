#### ❌ Q9 错题讲解：Traceability 的真正价值

题干：

> What is the **main benefit** of maintaining requirement traceability?

你选了 **A（It prevents scope changes completely）**，  
但实际上 **scope changes（范围变更）无法被完全阻止**，

> It links each requirement to its test cases and design elements.

## 第 3 步：把「流程」映射到 Use Case & Diagram

- **Normal flow（主成功路径）**：Scan → Pop-up → View Sorting Info → Earn Points → Done
    
- **Alternative flow（部分成功）**：Scan 成功，但用户关闭弹窗返回继续拍；或网络慢 → loading → 仍可完成
    
- **Exception flow（失败）**：相机权限拒绝 / AI 超时 / 无法取数 → 显示错误并结束
    

> 记住：**图**（Use Case Diagram）只画“关系”（actor↔use case、use case↔use case 的 include/extend），  
> **行为细节**写进 **Use Case Description**（Normal/Alternative/Exception flows），再由 **SRS** 一条条支撑。

## 🧩 Scenario 1 – “Waste Scanning and Detection”

A user opens the _Smart Waste Sorting Assistant_ app and taps **“Scan Waste.”**  
The system activates the camera, analyses the image with the AI model, and shows the detected waste type.  
If the waste cannot be recognized, the system displays “Unable to identify waste” and lets the user manually choose a category.

### Multiple-choice questions

1️⃣ Which event is the **Trigger** for this use case?   a
A. The system displays the detection pop-up.  
B. The user opens the app.  
C. The user taps “Scan Waste.”  
D. The AI model finishes analysis.

#### ✅ **Q1**

**题目：** Which event is the Trigger for this use case?  
**你的答案：A（The system displays the detection pop-up）**
思路：user do "sth"
❌ **正确答案：C – The user taps “Scan Waste.”**

#### 📘 讲解：

**Trigger** 是「引发用例开始的动作」。

- “User taps ‘Scan Waste’” → 用户主动启动系统 → ✅ 正确的触发条件。
    
- “System displays pop-up” → 这是**系统反应**，属于 Normal Flow 的中间步骤。
    

> 🧠 **记忆口诀：**  
> “Trigger starts the use case, not what happens after it starts.”

2️⃣ When the AI fails to recognize the waste but the user selects a category manually, which flow does this represent?   c -b
A. Normal Flow B. Alternative Flow C. Exception Flow D. Post-condition

#### ✅ **Q2**

**题目：** When AI fails to recognize the waste but user selects manually, which flow?  
**你的答案：C（Exception Flow）**

❌ **正确答案：B – Alternative Flow**

#### 📘 讲解：

**Alternative Flow** 表示「部分成功」或「通过不同路径完成目标」。  
在此情境中：

- 虽然 AI 识别失败，但用户**仍能手动选择分类并完成任务** → 属于成功路径的变体。
    
- **Exception Flow** 则是「彻底失败」，如 AI 崩溃、服务器断线等。

| Flow 类型     | 是否达成目标        | 示例           |
| ----------- | ------------- | ------------ |
| Normal      | ✅ 成功（AI 识别正常） | 显示结果、用户查看指南  |
| Alternative | ✅ 成功（路径不同）    | 用户手动选择垃圾类型   |
| Exception   | ❌ 失败          | 无法连接服务器、程序崩溃 |
|             |               |              |

> 🧠 **口诀：**  
> _Alternative Flow = “Different way to succeed.”_  
> _Exception Flow = “Failure, cannot proceed.”_

## ⚙️ Scenario 2 – “Login and Connection Error”

A student logs in with username and password.  
If the password is incorrect, the system allows another attempt.  
If the database connection is lost, the system shows “Cannot connect to server” and stops the process.

### Concept-definition questions

3️⃣ Define in one sentence what an **Alternative Flow** is within a Use Case Description.
**Alternative Flow**:when the password is correct,the system will allow users to detect the rubbishses for attempt

#### ❌ **Q3**

你的回答：

> “when the password is correct,the system will allow users to detect the rubbishes for attempt”

这一句语义和上下文都偏离主题。

#### ✅ 正确表达应该是：

> **An Alternative Flow** is a sub-case of the normal flow where the goal is still achieved, but through a different path or condition.

**例句：**

> For example, if the user enters the wrong password once but logs in successfully on the second attempt.

4️⃣ Define in one sentence what an **Exception Flow** is and how its result differs from an Alternative Flow.
Exception Flow**:when the password is incorrect,the system will prompt users to detect the item again

#### ❌ **Q4**

你的回答：

> “when the password is incorrect,the system will prompt users to detect the item again”

逻辑上错误：这里描述的其实仍是 _Alternative Flow_。  
**Exception Flow** 是指系统无法继续执行、失败结束。

#### ✅ 正确表达：

> **An Exception Flow** describes a failed case where the use case cannot achieve its goal due to an unexpected error or system failure.

**例句：**

> For example, when the system cannot connect to the database and the login process stops.

---
## 🧠 补充总结：三类 Flow 的区分口诀

|Flow 类型|含义|是否达成目标|典型例子|
|---|---|---|---|
|**Normal Flow**|主成功路径|✅|登录成功、扫描识别成功|
|**Alternative Flow**|次成功路径，部分成功|✅|用户手动分类、重新输入密码|
|**Exception Flow**|异常失败路径|❌|网络断开、数据库无法连接|

trigger:user sorts waste,Precondition:the GPS permission is ok. 
#### **Q5**

你的答案：

> Trigger: user sorts waste  
> Precondition: the GPS permission is ok

补充例句（考试时可这样写）：

> _Trigger:_ The user finishes sorting and confirms submission.

6️⃣ Compare **Alternative Flow** vs **Exception Flow** using this scenario.

**Alternative Flow**：the result is successfull ,"If GPS permission is denied, the system shows a message and allows manual location input.",**Exception Flow**:the result is fail

> **Alternative Flow**：success, “If GPS permission is denied, the system shows a message and allows manual location input.”

## 🧠 Scenario 5 – “Requirement Validation”

The project team reviews the requirement:

> “The system shall recognize waste images quickly.”  
> The reviewer states it cannot be verified because no time limit is defined.

### Concept-definition questions

9️⃣ What is the **Fit Criterion**, and why is it important for requirement validation?
Fit Criterion is the best SRS for to provide the function.It while it could fetch the requirement of system
你的答案：

> Fit Criterion is the best SRS for to provide the function.It while it could fetch the requirement of system.

❌ **不够准确。**

✅ **标准答案表达：**

> A _Fit Criterion_ defines a **measurable standard** that shows whether a requirement has been met.  
> It makes the requirement **testable and verifiable** during validation.

📘 **课件关键句（Week 13 V&V）：**

> “A Fit Criterion converts a vague statement into something measurable.”

🔟 Rewrite the above requirement so that it becomes **measurable** and **testable** according to the Quality Gateway Checklist.
The system shall recognize waste images within 5s and reduce the time waste.
upload the reviewer states it can be verified within limitted time.

你的答案：

> “The system shall recognize waste images within 5 s and reduce the time waste.”

✅ **方向正确**，但需语法更自然、指标更明确。

✅ **优化版写法：**

> _The system shall recognize waste images within 5 seconds under normal network conditions, with an accuracy of at least 90%._

→ 现在既 **measurable**（5 seconds, 90%）又 **testable**（QA team 可验证）。

### 📘 讲解：

当用户**拒绝相机权限**时，系统会提示“Camera access required”。  
这是一个**替代路径（Alternative Flow）**，因为主成功路径无法继续，但系统提供了**其他操作或返回**选项。

|Flow 类型|描述|结果|
|---|---|---|
|**Normal Flow**|用户同意权限 → 正常扫描|✅ 成功|
|**Alternative Flow**|用户拒绝权限 → 系统提示并返回|⚠ 可恢复|
|**Exception Flow**|相机模块崩溃或应用出错|❌ 失败结束|

🧠 **记忆口诀：**

> Alternative = “still can continue”; Exception = “must stop”.
