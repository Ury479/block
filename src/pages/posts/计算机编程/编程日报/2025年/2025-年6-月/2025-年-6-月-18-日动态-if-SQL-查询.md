#### 课程笔记 
##### `<where>` 关键字
xml 文件 `<where>` 关键字 去除 mysql 语句首行多余的`and`和`or` ，否则容易报错：
```
You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near 'and gender = 1 ... order by update_time desc'
```
测试语法：
```
//更新员工  
@Test  
public void testUpdate() {  
    //构造员工对象  
    Emp emp = new Emp(); // 创建一个新的 Emp 对象  
    emp.setId(18); // 设置员工的 ID 为 18。这是非常重要的，因为更新操作通常会根据 ID 来确定要修改哪条记录。  
    emp.setUsername("Tom1"); // 设置新的用户名  
    emp.setName("汤姆1"); // 设置新的姓名  
    emp.setImage("1.jpg"); // 设置新的图片URL  
    emp.setGender((short)1); // 设置新的性别（这里是男性）  
    emp.setJob((short)1); // 设置新的职位  
    emp.setEntrydate(LocalDate.of(2000, 1, 1)); // 设置新的入职日期为 2000年1月1日  
    emp.setUpdateTime(LocalDateTime.now()); // 设置更新时间为当前时间  
    emp.setDeptId(1); // 设置新的部门ID  
  
    //执行更新员工操作  
    empMapper.update(emp); // 调用 empMapper 的 update 方法，传入构造好的 Emp 对象，执行数据库更新操作  
}
```
注意使用 MybatisX 插件 xml 文件进行定位和跳转

##### `<set>` 关键字
用来去掉 sql 语句中多余的`,` 避免 mybatis 语法报错。

##### `<foreach>` 关键字 
循环遍历，用于批量删除数据库中元素：
```
<delete id="deleteByIds">  
    delete from mybatis.emp where emp.id in   
<foreach collection="ids" item="id" separator="," open="(" close=")">  
         #{id}  
     </foreach>  
</delete>
```

##### `<sql>,<include>`关键字
语法：
```
<sql id="commonSelect">  
  ……
</sql>
……
<select id="list" resultType="com.ury.mybatis_curd.pojo.Emp">  
    <include refid="commonSelect"/>  
    <where>  
        <if test="name != null">  
            name like concat('%', #{name}, '%')  
        </if>  
        <if test="gender != null">  
            and gender = #{gender}  
        </if>  
        <if test="begin != null">  
            and entrydate between #{begin}  
            and #{end}        </if>  
    </where>  
    order by update_time desc  
</select>
```

#### 年终总结：明确为领导做了哪些事情，拿到了什么样的结果
年终述职的关键点：

1. 给谁讲
2. 讲什么
3. 如何讲

首先最重要的是要明确「给谁讲」。

有的公司年终述职是给自己的直属领导，有的公司则是给更高一级领导甚至部门的负责人，针对不同的对象，要针对性的准备内容。

如果是给直属领导汇报，要围绕直属领导的年度目标汇报，多讲自己做了什么帮助他的目标达成；如果是更高级别领导甚至负责人，则要更多讲自己主动、自发做的事情，达成了什么结果、有什么思考。

展开一下「讲什么」。

对于直属领导来说，你就是他的资源，如果你一年下来为他做了很多事情，哪怕最后成果没有很多，在情感上他也是认可你的，久而久之就会成为所谓的“嫡系”，自然绩效不会差；反过来讲，你汇报的工作和他没有什么关系，就算很辛苦从 0 到 1，在领导那边也不会有很大的认可。

而对于更高一级的领导来说，你是他隔层的下属，他想知道的是你的能力如何、有没有做出一些匹配你薪资、级别的事情。所以你要讲的是主要由你完成、且有成果的事，这些事情可以体现你的技术、态度和团队协作能力。

最后说一下「怎么讲」。讲的时候，遵循 STAR 法则： 1.Situation(情景)：在什么背景下你开始的这个项目 2.Task(任务)：你承担了什么任务，有什么挑战 3.Action(行动)：为了达成目标你做了什么（可以适当讲讲加班加点，领导爱看这种） 4.Result(结果)：这个项目上线后取得了什么成果，这个非常重要
