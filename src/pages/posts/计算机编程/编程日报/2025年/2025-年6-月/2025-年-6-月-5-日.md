问题所在：

在 Vue 的单文件组件（SFC）中，你不能同时存在 <script setup> 和另一个普通的 <script> 标签（带有 export default）。

- <script setup> 是 Vue 3 提供的语法糖，用于在使用 Composition API 时简化代码，它会自动将顶层声明（变量、函数、导入等）暴露给模板使用。

- 普通的 <script> 标签（带有 export default）是 Options API 或不使用 <script setup> 时使用 Composition API 的方式。

这两种 script 标签的用法是互斥的。同时存在会导致 Vue 无法正确解析组件的 setup 逻辑和 options 选项，从而可能引起组件注册、模板编译或渲染的错误，表现出来就是组件不显示或报错。

你之前看到的 [Vue warn]: resolveComponent can only be used in render() or setup(). 错误也印证了这一点，Vue 在解析组件时遇到了结构问题


这个错误 no-unused-vars 的意思是，你在 /Users/ury/Documents/project/MyProject/javaWeb/day003_vue_2/src/App.vue 文件的第 37 行第 7 列定义了一个名为 changeMessage 的变量或函数，但在代码的其他地方没有使用它。

在 Vue 3 的 <script setup> 中，如果你定义了一个变量或函数但没有在 <template> 中使用，ESLint 会将其标记为未使用的变量，从而报错。

### 解决方案：

你有两种选择来解决这个错误：

1. 使用 changeMessage 函数： 如果你打算在界面上通过某个交互（比如点击按钮）来触发 changeMessage 函数，你需要在 <template> 中调用它。例如，给一个按钮添加 @click="changeMessage" 事件。

2. 移除 changeMessage 函数： 如果你不再需要 changeMessage 函数的功能，直接将它从 <script setup> 中删除即可。
