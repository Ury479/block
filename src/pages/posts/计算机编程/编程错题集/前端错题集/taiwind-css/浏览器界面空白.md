# `template` 编写有误
### **你有一个小 bug：**

`<span>@{{event.time}} on {{event.date}}}</span>`

应改为：

vue

CopyEdit

`<span>@{{ event.time }} on {{ event.date }}</span>`

否则 `{{event.date}}}` 右边多了一个 `}` 会导致解析错误。

# `<script setup lang="ts">` 代码**缺失
你提到的这段 `<script setup lang="ts">` 代码如果**缺失**，**确实会导致页面卡片不显示（界面空白）**，原因如下：

---

### ❗ 原因说明

在 Vue 3 的 `<script setup>` 语法中：

```ts
defineProps<{ event: Event }>()
```

这段代码是必须的，它告诉组件：

> “我从父组件接收了一个名为 `event` 的 prop，类型为 `Event`”。

---

### 🔥 如果这段代码缺失：

- 你在 `<template>` 中写了很多 `{{ event.title }}` 等；
    
- 但 Vue 不知道 `event` 是谁，也没有任何默认值或定义；
    
- 于是 `event` 被视为 “未定义变量”，Vue 报错；
    
- 最终结果是：**页面渲染失败，卡片不会显示**，控制台一般会有报错类似：
    

```bash
[Vue warn]: Unhandled error during execution of render function
ReferenceError: event is not defined
```

---

### ✅ 正确做法

确保你组件内有这段代码：

```ts
<script setup lang="ts">
import type { Event } from '@/types'

defineProps<{
  event: Event
}>()
</script>
```

并确认你的 `EventCard.vue` 是这样被调用的：

```vue
<EventCard :event="event" />
```

---

### 🧪 调试建议

1. 浏览器控制台（Console）是否有 `event is not defined` 错误？
    
2. 确保组件文件名和传入 prop 匹配。
    
3. 可以在 `<template>` 顶部加个 `{{ event }}` 做快速排查。
    

---

结论：**是的**，`defineProps` 缺失会导致空白页面，你需要添加回来才能正常显示卡片。添加后记得保存文件并重新运行 `npm run dev`。
