## ip 地址登录不了
### nginx 主页面 error
关键症状是：
- `curl http://127.0.0.1:8080/user/current` 能返回业务 JSON（说明 Spring Boot 在跑）
- `curl http://127.0.0.1/user/current` 返回 Nginx 404（说明 Nginx 没把 `/user/*` 转发到 8080）
- 浏览器看到 “nginx error / not found / Welcome” 类页面（说明 80 端口提供的是默认站点或错误 server 块）
    
结论：**当前主要卡在 Nginx/宝塔站点反向代理未生效（或配置未被当前 Nginx 加载）**，不是 Maven。
****
策略：
1. 检查服务器安全组
2. 检查宝塔 linux java项目启动指令
3. 检查 mysql 宝塔面板配置情况

### 反向代理
![Pasted image 20251227112539.png](/media/6ed01f6b1e352cce.png)conf 文件被意外删除，需要重新添加站点，**代理目录：**`http://127.0.0.1:8080`(不是 ip 地址，否则就是自己代理自己)，教程使用静态网页，需要调整 Nginx 默认反向代理文件，要求完整替代版，否则会报错，需要同时修改反向代理文件和配置文件，更新 dist 目录，删除无关文件

方案：**conf 丢失，立马删 dist 文件**,进入 dist,引入所有文件
