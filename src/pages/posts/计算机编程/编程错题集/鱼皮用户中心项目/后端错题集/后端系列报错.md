# 🟥 **问题 1：mock（json-server）与真实后端冲突**

### ❌ 现象

- 前端 `/user/current` 总是返回固定 “测试用户”
    
- 登录后 GlobalHeader 仍显示“未登录”
    
- userRole 类型变成 string（"admin" / "test"）
    
- 与真实 Spring Boot 后端完全脱节
    

### ❌ 根因

你一开始使用 mock server：

```
http://localhost:3000/user/current
```

mock 返回的结构是：

```json
{
  "id": 1,
  "userRole": "admin"
}
```

→ userRole 是 string  
→ 干扰了前端类型判断，导致报错。

更严重的是：mock 会覆盖真实后端 session 登录态导致：

- 登录后 fetchLoginUser() 仍取到 mock 数据
    
- 后端明明已登录，但前端一直认为未登录
    

### ✅ 最终解决方案

- **彻底删除 mock 文件夹**
    
- 删除 `/api → 3000` 的代理
    
- 所有接口统一转发到 **[http://localhost:8080](http://localhost:8080/)**
    

---

# 🟥 **问题 2：Vue + TypeScript 报错

`Type string is not assignable to type number`**

### ❌ 现象

Pinia store 报错：

```ts
userRole: "",
userRole: "test",
```

报错：

```
Type string is not assignable to type number | undefined
```

### ❌ 根因

mock 数据返回的是 `"admin"`  
你把 userRole 错误写成了 string。

但真实后端字段：

```java
private Integer userRole; // number
```

### ✅ 解决方案

Pinia 改为：

```ts
userRole: 0;
```

模拟用户也改为：

```ts
userRole: 0;
```

并强制类型为 number:

```ts
export interface CurrentUser {
  userRole?: number;
}
```

# 🟥 **问题 3：前端 axios 的返回解析逻辑与后端不一致**

### ❌ 现象

你在前端用了教程的解析方式：

```ts
if (res.data.code === 0 && res.data.data)
```

但你的后端返回结构是：

```
User
null
```

没有 code/data/message

### ❌ 根因

教程用 BaseResponse  
你用的是最简单的返回用户对象

### ✅ 解决方案

改为解析 User：

```ts
const user = res.data;
if (user && user.id) { ... }
```

sync:

- UserLoginPage.vue 处理 login 结果要改
    
- loginUserStore.fetchLoginUser 处理 /user/current 要改

# 🟥 **问题 4：前端 devServer 的代理配置写错（嵌套 /api）**

你的代码：

```js
"/api": {
    target: "http://localhost:3000",
    ...
    "/api": {
         target: "http://localhost:8080",
    }
}
```

### ❌ 根因

你把 `/api` 代理嵌套写了两层 → 完全无效  
真正生效的 target 反而是 mock 端口 3000

### ❌ 后果

前端所有接口都走 mock，而不是后端  
这也是为什么你登录永远不成功

### ✅ 解决方案

正确的 vue.config.js：

```js
devServer: {
  proxy: {
    "/api": {
      target: "http://localhost:8080",
      changeOrigin: true,
      pathRewrite: { "^/api": "" }
    }
  }
}
```

# 🟥 **问题 5：UserManagePage 使用假数据，导致字段不一致**

### ❌ 现象

table 代码：

```ts
data = [
  { name: "John", tags: ["cool"] }
]
```

但后端字段：

```
id, username, userAccount, userRole, createTime
```

### ❌ 后果

表格展示不正确，删除功能无法完成。

### ✅ 解决方案

1. 删除所有 demo 假数据
    
2. 使用：
    

```ts
const data = ref([]);

const loadData = async () => {
  const res = await searchUsers("");
  data.value = res.data;
};
```

3. 添加删除按钮：
    

```vue
<a @click="onDelete(record.id)">删除</a>
```

# 🟥 **问题 7：登录后 GlobalHeader 不更新用户名**

### ❌ 根因（两个）

1. mock → 永远返回测试用户
    
2. fetchLoginUser() 按错误格式解析后端数据
    
3. vue.config.js 代理错误 → 前端请求没打到后端
    

### ✅ 解决方案

- 去掉 mock

- 修复 vue.config.js 代理
    
- 登录之后调用 `await loginUserStore.fetchLoginUser()`

# 🟥 **问题 8：UserController 少了 /user/current**

### ❌ 后果

前端无法判断登录态  
GlobalHeader 永远显示 “未登录”

### ✅ 解决方案

添加：

```java
@GetMapping("/current")
public User getCurrentUser(HttpServletRequest request) {
    Object userObj = request.getSession().getAttribute(UserService.USER_LOGIN_STATE);
    return userObj == null ? null : (User) userObj;
}
```

完全解决 Pinia 读取登录态的问题。
