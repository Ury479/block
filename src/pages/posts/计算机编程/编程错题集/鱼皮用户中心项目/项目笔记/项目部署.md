## 宝塔linux 配置:不涉及容器
1. 登录阿里云服务器，配置 8080 安全组，登录控制台命令行，用 https
	1. 检查 mysql 情况，确认数据库密码是 user_center
	2. 宝塔 mysql 和 docker mysql 不互通
	3. 宝塔 linux 服务器端口 8888，没有安全组配置
2. 服务器远程登录 VNC
	1. 前端：导入服务器 ip
	2. 后端导入jar，前端导入 dist，前端 ph无误后再导入服务器 java 项目
	3. **server{} 放站点配置；proxy 片段只放反向代理配置 location **
	4. 每次项目更新必须重新导入文件，重启端口
	5. 确认网址 ip 正常打开后再考虑反向代理

## docker 容器化部署：微信云托管
#### docker下载
1. 宝塔面板：docker 管理，3.9
2. 参考现成Dockerfile 脚本，无需刻意理解，先部署后端，再部署前端
3. 上线 github 仓库，服务器拉取，使用 CI/CD,对接 devops
4. 多阶段构建：降低容器体积

#### 微信云托管
1. 前端：导入 dist,Dockerfile,docker 3 个目录，压缩文件
2. 后端：target 目录，Dockerfile 文件，上传服务器公网地址(阿里云)，配置微信云托管数据库，注意新建项目，**引入github 仓库地址和 Dockerfile 文件**，平台会自动部署，需要核对 dockerfile 源码，上传版本控制软件(Fork),让 docker 进行编译
3. 微信云托管自动提供网址，可视化分析日志，不用登录服务器看日志

##### AI 提示词
 启用微信云托管进行项目部署，请帮我核查**前端请求基址**：`src/request.ts` 有关 axios 的配置，处理反向代理问题，检验Dockerfile 写入的 nginx `default.conf`，考虑 405 报错的情况，处理生产环境请求打到了“前端域名的 /user/login”的问题，重新配置微信云托管上传的文件和数据库信息，确保`/api/user/login`  被 nginx 的 location /api/ 正确代理到后端，核查nginx 里 /api/ location 是否生效，proxy_pass 写法是否正确，避免路径拼接异常，指导我分别验证平台网关路径规则，和浏览器走的反向代理，避免API 请求打到前端静态站点路径 `/user/login`或请求没到正确后端、被前端/网关拦截中断
