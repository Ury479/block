- 智能体=大模型+工具
- 提示词是为了引导大模型给出相应的回复，需要靠限制条件保护系统安全，缺乏约束条件会导致 AI 跑偏，消耗 tocken 额度
- Dify 启动之前必须先部署 docker,获取 url，进行文档切块和向量化，利用工具检索互联网上信息，然后交给大模型整理
- 提示词生成器本身需要大模型进行支持，安全审查同样可依靠大模型进行实现从而实现集成，提示词本身需要限制输出格式，否则可能导致混乱和乱码，提示词本身可以设计变量，让大模型引用知识库的内容，**知识库是核心**
- Dify 中提示词生成需要通过大模型进行优化，提示词涵盖系统提示词 advisor 和用户提示词，智能体分类涵盖了 embedding,rank 等助手工具
- RAG 负责解决大模型幻觉问题，在 LLM 对话过程中导入上下文，用瑞达利欧的思想，**将上下文信息导入 LLM，逐步构建决策所需的上下文，形成决策算法**
- Dify:插件本身就是一个适配器，把任意 REST API 封装成 Dify Tool并让大模型进行调用，可能涉及 python 代码，
- 节点思想，找到节点所需的代码
- 工作流是业务逻辑（OCI）的可视化执行
- dify 工作流意图识别模块需要配置代码，知识库检索需要引入知识库，并配置输出端口，系统提示词需要在输入框中代码实现，一般工作流涉及多个大模型的编排，
- 意图识别部分涉及用户输入信息的归类和用户意图的过滤，普通问题让大模型直接回答，关键问题对应知识库
- dify 智能体编排可以用OCI 的方式进行实现，不同 LLM 负责不同环节，将职能压缩到大模型的范围内，同时设置过滤机制，关注 dify 嵌入网页应用和 Agents 嵌入 API 应用，靠代码进行调用，OCI 的本质就是对工作流的拆解和封装，dify 嵌入网页可以直接在 Zcode 中完成，将页面嵌入到合适位置 
- postman 能够验证接口地址是否联通，进行 API 接口测试
