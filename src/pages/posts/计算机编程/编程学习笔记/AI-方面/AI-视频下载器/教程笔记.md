### 产品思路
1. UI 设计：免费项目也要以尽可能吸引付费为导向，用户思维
2. 前期轻后端，轻数据库，后续接入
3. 开发阶段在前人的肩膀上进行，拒绝重复造轮子
4. 前端界面照抄：Screenshot to code,zcode,codex；照搬付费模式，外网发布赚钱
### 提示词思路
1. 尽量吸引人付费的设计，必须 browser use 浏览器访问，理解对标网站的 UI 设计
2. 理解开源项目，在别人的基础上进行封装或直接二开
3. 提示词需包含人工验证和测试环节，让 AI 先完成核心功能后上线，之后再开新需求
4. 让 AI 沉淀需求分析文档和方案设计文档
5. AI 罗列任务不要贪心，优先聚焦核心功能实现，自主修复所有报错，跑通核心业务流程
6. 知道相应概念才能针对性改掉相应 bug
7. 关注 AGENTS.md 文档编写，精简核心，每次报错解决后沉淀入全局项目文档
8. 核心功能实现后要整理总结文档，AI 梳理，提交 git，确保进入下一轮开发所需的信息源中，让 AI 了解前置项目的背景
9. 对标竞品，利用 AI 逆向功能实现，让 AI 直接给出结果，坐享其成。当 AI 陷入瓶颈时，需要中断并人工干预，给出新的思路让 AI 进行参照。
```
## 限制条件
 1.项目核心功能闭环后，封装为可复用的模块，保持开闭原则，开放 API 接口，只添加新功能，不修改模块内代码，
 2.使用门面模式等设计模式精简重复代码，确保高内聚，低耦合，
 3.每开发新的功能均进行回归测试，自主修复所有报错，跑通核心业务流程再添加功能，
 4.前端 UI 界面升级后改成更容易吸引付费的样式，启用 firecraw MCP 和 context7 调研外网现有的产品，罗列 md 清单，并启用browser use 功能登录并借鉴UI 样式，分析其付费模式任务完成后抽取 skills
```
