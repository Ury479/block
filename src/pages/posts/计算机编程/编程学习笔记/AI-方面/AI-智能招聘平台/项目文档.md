# 一、项目概述

- **项目名称**：AI 智能体招聘平台
    
- **平台类型**：Web 前后端分离系统
    
- **核心能力**：
    
    - 支持候选人 **简历上传 & AI 解析**
        
    - 支持职位 **检索 / 筛选 / AI 匹配推荐**
        
    - 提供 **用户登录 / 用户管理 / 用户详情** 等后台管理功能
        
    - 后端集成 GPT 能力，作为 **智能简历解析 + 智能推荐引擎**
        

---

# 二、开发环境与基础工具

## 2.1 开发设备与操作系统

- **开发设备**：Apple Mac（Intel / Apple Silicon 均可）
    
- **操作系统**：macOS 13+（Ventura 及以上版本推荐）
    

## 2.2 基础环境

- **Node.js**：20.x（稳定版）
    
    - 包管理器：npm（默认），可根据团队习惯使用 pnpm / yarn
        
- **JDK**：Java 17（Spring Boot 3 官方推荐）
    
- **MySQL**：8.x（开发环境可使用本地或 Docker 容器）
    
- **Docker**：
    
    - Docker Engine / Docker Desktop（用于前后端容器化部署）
        
- **Nginx**：
    
    - 以 Docker 镜像形式使用，用于前端静态资源托管 + 反向代理后端 API
        

## 2.3 IDE 与开发工具

- **后端 IDE**：IntelliJ IDEA Ultimate
    
- **前端 IDE**：WebStorm
    
- **版本控制**：
    
    - Git（命令行）
        
    - Fork（图形化 Git 客户端，便于查看分支 / 提交历史 / PR）
        
- **接口调试与文档**：
    
    - Apidog（接口调用、Mock、文档管理）

# 三、前端技术栈（React + Vite + Zustand）

前端主要用于实现简历上传页、岗位检索页、岗位筛选页、用户登录页、用户管理页、用户详情页等。

## 3.1 技术选型

- **框架**：React 18（函数式组件 + Hooks）
    
- **构建工具**：Vite（开发热更新 + 生产构建）
    
- **语言**：TypeScript
    
- **路由**：React Router v6
    
- **全局状态管理**：Zustand
    
    - 用于管理：登录状态、用户信息、职位列表、筛选条件、全局 UI 状态等
        
- **HTTP 客户端**：axios
    
    - 统一封装 baseURL、请求/响应拦截器、JWT 注入、错误处理
        
- **认证方式**：JWT（前端通过 `Authorization: Bearer <token>` 携带）
    
- **UI / 样式**：
    
    - 自定义 **Base44 CSS 设计体系**：
        
        - 包含统一的颜色、间距、字体大小、圆角、按钮、输入框、卡片等组件风格
            
    - 可以在 `styles/base44.css` 中集中管理
        
    - 通过封装基础组件 `<B44Button />`, `<B44Input />`, `<B44Card />` 等在业务中复用
        
- **表单 & 校验**（可选）：
    
    - React Hook Form / Formik（可选，用于复杂表单）
        
- **打包输出**：
    
    - 构建产物为静态文件（HTML+JS+CSS），通过 Nginx 托管
        

## 3.2 前端主要依赖（npm）

运行时依赖（dependencies）示例：

- （可选）`react-hook-form` / `formik` / `yup` 等

# 四、后端技术栈（Spring Boot 3 + JPA + Security + JWT）

后端主要提供用户认证、职位管理、简历管理、AI 智能服务（简历解析、职位匹配）、接口权限控制等。

## 4.1 技术选型

- **核心框架**：Spring Boot 3.x
    
- **Web 层**：
    
    - Spring Web（Spring MVC）
        
    - 使用 `@RestController` 暴露 REST API
        
- **数据访问层**：
    
    - MySQL 8.x
        
    - Spring Data JPA（Repository 模式）
        
- **安全与认证**：
    
    - Spring Security 6
        
    - JWT（登录成功后签发 Access Token）
        
    - 自定义过滤器，实现：
        
        - 请求头解析 JWT
            
        - Token 校验、用户认证上下文构建
            
- **实体与样板代码**：
    
    - 使用 **Lombok** 简化实体类、DTO、Service 等样板代码
        
    - 示例注解：`@Getter`、`@Setter`、`@Builder`、`@Data`、`@NoArgsConstructor`、`@AllArgsConstructor`
        
- **参数校验**：
    
    - Jakarta Bean Validation（`spring-boot-starter-validation`）
        
    - 在 DTO 上使用 `@NotNull`、`@Size` 等注解
        
- **AI 能力集成**：
    
    - 使用 GPT（例如通过 HTTP 调用 OpenAI / 兼容 API）
        
    - 典型服务：
        
        - `AiResumeService`：简历文本解析成结构化数据
            
        - `AiMatchService`：候选人简历与职位 JD 匹配评分与解释
            
- **对接 Firebase**：
    
    - Firebase Storage 作为简历文件存储
        
    - 后端负责：
        
        - 接收前端传来的文件 / 上传凭证
            
        - 调用 Firebase SDK 或 REST API 上传文件
            
        - 将文件 URL 存入 MySQL
            
- **日志与监控**：
    
    - Spring Boot 自带日志（Logback）
        
    - 后续可接 ELK / Loki / Prometheus 等（可选）

# 五、基础设施与部署（Nginx + Docker）

## 5.1 容器化策略

- **前端容器**：
    
    - 使用 Node 20 构建 React/Vite 产物
        
    - 使用 Nginx 容器托管静态文件
        
    - Nginx 同时负责反向代理 `/api` 到后端容器
        
- **后端容器**：
    
    - 基于 OpenJDK 17 镜像
        
    - 运行 Spring Boot 可执行 jar
        
- **数据库**：
    
    - MySQL 可以单独使用 Docker 容器
        
- **部署平台**：
    
    - 阿里云 ECS / 阿里云容器服务（ACK）
        
    - 通过 Git + Fork 管理代码，通过 Docker 镜像发布
        

## 5.2 Docker 相关（简要）

- 前端 Dockerfile：
    
    - Stage 1：Node 20 构建 `npm run build`
        
    - Stage 2：Nginx 托管 `/usr/share/nginx/html`
        
- 后端 Dockerfile：
    
    - 使用 `eclipse-temurin:17-jdk` 或类似镜像
        
    - 拷贝 jar → `ENTRYPOINT ["java","-jar","/app/app.jar"]`
        
- docker-compose：
    
    - 定义 `frontend`、`backend`、`mysql` 等服务
        
    - Nginx 中配置 `location /api/ { proxy_pass http://backend:8080/; }`

# 六、版本控制与协作（Git + Fork）

## 6.1 仓库划分

- 方式一：前后端同仓库（monorepo）
    
    - `frontend/` + `backend/`
        
- 方式二：前后端分仓库，按团队职责分配
    

## 6.2 分支策略（简化）

- `main`：稳定可发布版本
    
- `dev`：日常开发整合
    
- 特性分支：`feature/xxx`（2 人分别创建自己的功能分支）
    
- 使用 **Fork**：
    
    - 直观查看提交历史
        
    - 对比分支差异
        
    - 做 PR / Merge Review

# 七、总结

- **开发设备**：macOS + Node.js 20 + JDK 17 + Docker
    
- **前端**：React + Vite + TypeScript + React Router + Zustand + axios + Base44 CSS 自定义 UI
    
- **后端**：Spring Boot 3 + Spring Web + Spring Data JPA + Spring Security + JWT + Lombok + MySQL
