## 消息调用
```js
export default class Search extends Component {


/*

search 方法 —— 点击搜索按钮时触发

用箭头函数定义，保证 this 指向组件实例

*/

search = ()=>{

/*

获取用户输入 —— 连续解构赋值 + 重命名

等价于：

const keyWordElement = this.keyWordElement // 拿到 input DOM

const keyWord = keyWordElement.value // 拿到输入值

一步写法：const { keyWordElement: { value: keyWord } } = this

this.keyWordElement 是通过下面 render 中的 ref 回调绑定的：

ref={c => this.keyWordElement = c}

React 会在渲染时把 input DOM 节点传给回调参数 c

*/

const {keyWordElement:{value:keyWord}} = this

  

// 发送请求前：发布消息通知 List 更新状态

// isFirst: false → 不再显示欢迎语

// isLoading: true → 显示加载中

PubSub.publish('atguigu',{isFirst:false,isloading:true})

  

/*

发送网络请求

- 请求地址：/api1/search/users?q=关键词

- /api1 会被 setupProxy.js 代理到 https://api.github.com

- 实际请求：https://api.github.com/search/users?q=关键词

axios.get() 返回 Promise，用 .then(成功回调, 失败回调) 处理

*/

axios.get('/api1/search/users?q=${keyWord}').then(

// 请求成功：引用PubSub.publish 发布消息，关闭 loading，传递用户列表

response =>{PubSub.publish(`atguigu`,{isLoading:false,users:response.data.items})},

  

// 请求失败：发布消息，关闭 loading，传递错误信息

error => {PubSub.publish(`atguigu`,{isLoading:false,err:error.message})}

)
```

都是调用 **`PubSub.publish()`** 发布消息，通知 List 组件更新状态。

---

## 为什么要这么写？

Search 和 List 是**兄弟组件**，没有父子关系，无法直接传 props。PubSub 解决了这个问题：

```
Search（发布者）——→ PubSub 消息总线 ——→ List（订阅者）
   publish()                              subscribe()
```

三次发布对应搜索的三个阶段：

| 阶段 | 发布内容 | List 收到后做什么 |
|------|---------|-----------------|
| 点击搜索，请求发出前 | `{isFirst:false, isLoading:true}` | 隐藏欢迎语，显示 Loading |
| 请求成功 | `{isLoading:false, users:[...]}` | 隐藏 Loading，渲染用户卡片 |
| 请求失败 | `{isLoading:false, err:'错误信息'}` | 隐藏 Loading，显示错误提示 |

## 具体语法

### `PubSub.publish(消息名, 数据)`

```javascript
// 发布方（Search）—— 发送消息
PubSub.publish('atguigu', { key: value })
//                 ↑              ↑
//            消息频道名      传递的数据对象
```

### `PubSub.subscribe(消息名, 回调)` 

```javascript
// 订阅方（List）—— 接收消息
this.token = PubSub.subscribe('atguigu', (msg, stateObj) => {
//                                          ↑       ↑
//                                     消息名(不用)  收到的数据
    this.setState(stateObj)  // 直接用收到的对象更新 state
})
```

### 为什么数据格式是 `{isLoading:false, users:[...]}`？

因为 List 的 state 结构是：

```javascript
state = {
    users: [],
    isFirst: true,
    isLoading: false,
    err: ''
}
```

发布的对象直接传给 `this.setState()`，**键名必须和 state 一致**，React 会合并更新对应字段：

```javascript
// Search 发布
PubSub.publish('atguigu', {isLoading:false, users:response.data.items})

// List 收到后
this.setState({isLoading:false, users:response.data.items})
// 等价于：只更新 isLoading 和 users，isFirst 和 err 不变
```

## ⚠️ 当前代码有 2 个 bug

````jsx
// ...existing code...

		// bug1: isloading → isLoading（大写 L），否则 List 的 state 匹配不上
		PubSub.publish('atguigu',{isFirst:false, isLoading:true})

		// bug2: 普通引号 '...' 不支持 ${} 插值，必须用反引号 `...`
		axios.get(`/api1/search/users?q=${keyWord}`).then(
// ...existing code...
````

| Bug         | 原代码                       | 正确写法                          | 后果                                                  |
| ----------- | ------------------------- | ----------------------------- | --------------------------------------------------- |
| `isloading` | `isloading:true`          | `isLoading:true`              | List 的 state 里是 `isLoading`，小写 l 匹配不上，Loading 永远不显示 |
| 普通引号        | `'/api1/...q=${keyWord}'` | `` `/api1/...q=${keyWord}` `` | `${keyWord}` 不会被替换，请求参数永远是字面量 `${keyWord}`          |
|             |                           |                               |                                                     |
### react 组件
```html
<div>

{/*

ref 回调：React 渲染此 input 时，会调用 c => this.keyWordElement = c

把 input 的 DOM 节点保存到 this.keyWordElement

之后在 search() 方法中就能通过 this.keyWordElement.value 拿到用户输入

*/}

<input ref={c => this.keyWordElement = c} type='text' placeholder="输入关键词点击搜索"></input>

{/* 点击按钮 → 调用 this.search → 发请求 → 更新 App state */}

<button onClick={this.search}>搜索</button>

</div>
```
