路由类似函数，通过封装好的函数调用页面，实现页面切换，形式类似 vue
## React Router 路由组件详解

### 🎯 应用场景对比

| 应用类型 | 之前的GitHub搜索 | 引入路由后 |
|----------|------------------|------------|
| **页面结构** | 单页面：搜索栏+结果列表 | 多页面：首页/关于/用户详情等 |
| **URL变化** | 始终是 `localhost:3000` | `localhost:3000/home` `localhost:3000/about` |
| **组件切换** | 条件渲染（isFirst/isLoading） | 路由切换（URL驱动） |
| **用户体验** | 无法收藏特定搜索结果 | 可分享URL，浏览器前进后退 |

## 路由组件核心概念

### 1. **路由组件 vs 一般组件**

```jsx
// 一般组件（之前学的）
<Search updateAppState={this.updateAppState} />
<List users={users} isLoading={isLoading} />

// 路由组件（新概念）
<Route path="/demo" component={Demo} />
<Route path="/search" component={Search} />
```

| 特性 | 一般组件 | 路由组件 |
|------|----------|----------|
| **渲染方式** | 直接写标签 `<Search />` | 通过 Route 配置 |
| **显示时机** | 父组件决定 | URL 路径匹配时 |
| **props来源** | 父组件传递 | Router 自动注入 |
| **文件位置** | components | `src/pages/` |

## 实际应用场景举例

### 📱 将GitHub搜索改造为路由应用

````jsx
// 文件结构升级
src/
├── components/           # 一般组件（复用性强）
│   ├── SearchBox/       # 搜索输入框
│   └── UserCard/        # 用户卡片
├── pages/               # 路由组件（页面级）
│   ├── Home/            # 首页
│   ├── SearchResult/    # 搜索结果页
│   ├── UserDetail/      # 用户详情页
│   └── About/           # 关于页面
└── App.js               # 路由配置

// App.js - 路由配置
function App() {
  return (
    <BrowserRouter>
      <Header />
      <Switch>
        <Route exact path="/" component={Home} />
        <Route path="/search/:keyword" component={SearchResult} />
        <Route path="/user/:login" component={UserDetail} />
        <Route path="/about" component={About} />
      </Switch>
    </BrowserRouter>
  )
}

// 页面间跳转
function SearchBox() {
  const history = useHistory()
  
  const handleSearch = (keyword) => {
    // 跳转到搜索结果页，URL变为 /search/react
    history.push(`/search/${keyword}`)
  }
}

// 获取URL参数
function SearchResult() {
  const { keyword } = useParams() // 获取 :keyword 参数
  const location = useLocation()   // 获取查询字符串
  
  useEffect(() => {
    // 根据URL参数搜索
    fetchUsers(keyword)
  }, [keyword])
}
````

## 🔗 与之前知识的联系

| 之前学的 | 路由中的应用 | 进化点 |
|----------|-------------|--------|
| **props传递** | `history.push('/user', {userData})` | URL驱动，可分享 |
| **条件渲染** | `<Route>` 根据URL自动渲染 | 更语义化 |
| **PubSub通信** | 路由参数 + Context | 跨页面状态管理 |
| **生命周期** | `componentDidUpdate` 监听路由变化 | 响应URL变化 |

路由本质上是**将组件渲染从状态驱动改为URL驱动**，使应用更像传统网站，同时保持SPA的用户体验。
