## **1）深拷贝/浅拷贝混淆会引发的问题（典型后果）**

### **问题 A：改“副本”却把“原数据”也改了（最常见）**

**现象**

- 你做了一个编辑页（form），想在点“保存”前不影响列表数据
    
- 结果你在表单里改字段，列表/详情页也跟着变
    
- 点“取消”也回不去了
    

**根因**

- 你用的是浅拷贝：{...obj} / [...arr] 只复制第一层
    
- 嵌套对象/数组仍然共享引用
    

---

### **问题 B：历史记录/撤销（undo）失效**

  

**现象**

- 你想保存“上一次状态”用于撤销
    
- 结果撤销后发现内容没变，或变得很奇怪
    
**根因**

- 你保存的是同一个引用（浅拷贝或直接赋值）
    
- 后续修改把“历史快照”也污染了

---

### **问题 C：状态更新不符合预期（React/Vue 都可能遇到）**

**现象**

- UI 显示和你想的不一致，或者某些地方突然更新、某些地方不更新
    
- 组件/页面出现“莫名其妙联动”
    

**根因**

- 多个地方共享同一个嵌套引用，改了一处等于改了所有引用它的地方
    

---

### **问题 D：性能问题（过度深拷贝）**

  

**现象**

- 页面卡顿、更新变慢、频繁创建大对象导致内存增加
    

  

**根因**

- 不分场景把大对象全部深拷贝（例如每次输入都 structuredClone 一个巨大的 state）

## **2）浅拷贝/深拷贝的典型错误清单**

  

### **错误 1：以为** 

### **{...obj}**

###  **是“完全复制”**

```
const a = { profile: { city: 'CM' } }
const b = { ...a }
b.profile.city = 'BKK' // a.profile.city 也变了
```

### **错误 2：直接引用赋值当成复制**

```
const b = a // 不是复制，是同一个对象
```

### **错误 3：数组里放对象时只拷贝数组不拷贝元素对象**

```
const list2 = [...list] // list2 是新数组，但每个元素对象还是同一个
```

### **错误 4：JSON 深拷贝滥用/误用**

```
JSON.parse(JSON.stringify(obj))
```

可能导致：

- Date 变字符串
    
- undefined/function/Symbol 丢失
    
- 循环引用直接报错

## **3）**

## **this**

##  **指向混乱会引发的问题（典型后果）**
### **问题 A：点击事件一触发就报错**
**现象**

- 控制台：Cannot read properties of undefined (reading 'setState')
    
- 或者你打印 this 是 undefined
    

  

**根因**

- 类组件方法未绑定 this
    
- onClick={this.changeWeather} 传过去后，调用时丢失上下文
    

---

### **问题 B：state 更新失败或读不到 state**
**现象**
- 点击无反应
- 或者读 this.state 报错/是 undefined

**根因**

- this 不是组件实例 → this.state / this.setState 不存在
    
---

### **问题 C：把函数“调用了”而不是“传递”**

  

**现象**

- 页面一加载就执行函数，不是点击才执行
    
- render 里出现死循环或频繁执行
    

  

**错误写法**

```
onClick={this.changeWeather()} // 立刻调用
```

**正确写法**

```
onClick={this.changeWeather}   // 传函数引用
// 或 onClick={() => this.changeWeather()}
```

---

### **问题 D：在回调里拿旧 state（异步更新导致）**

  

**现象**

- 连点两下结果只变一次
    
- 状态切换不稳定
    

  

**根因**

- setState（React）可能是批量/异步
    
- 使用旧值计算新值时应使用函数式更新
    

  

**推荐写法**

```
this.setState(prev => ({ isHot: !prev.isHot }))
```

## **4）**

## **this**

##  **相关典型错误清单（类组件）**

  

### **错误 1：未 bind**

```
changeWeather() {
  this.setState(...)
}
// onClick={this.changeWeather} 会导致 this 丢失
```

### **错误 2：bind 写错或忘了写**

```
this.changeWeather = this.changeWeather.bind(this) // 正确
```

### **错误 3：箭头函数写法与普通方法混用理解错误**

- changeWeather = () => {}：this 自动正确（实例字段）
    
- changeWeather() {}：需要 bind 或者包装成箭头回调

## **5）两类问题叠加会出现什么“更隐蔽”的 bug？**

  

### **组合问题 1：this 修好了，但浅拷贝导致“串数据”**

  

**现象**

- 点击确实触发了 setState
    
- 但某个地方“原数据被改了”，导致别的组件也变化
    

  

**解释**

- 事件没问题，问题在共享引用
    

---

### **组合问题 2：深拷贝做得太重 + 频繁 setState → 卡顿**

  

**现象**

- 点击/输入明显延迟
    
- profiler 显示频繁大量对象创建
    

  

**解释**

- 每次更新都 deep clone 全量对象，成本过高
    

---

## **6）排错时最常用的 4 个判断口诀**

1. **“改副本原件也变”** → 先怀疑浅拷贝/共享引用
    
2. **“点击报 this undefined / setState undefined”** → 先怀疑 this 绑定
    
3. **“页面一进来就执行”** → 先检查 onClick={fn()}
    
4. **“连点结果不对”** → 用函数式 setState（prevState）
