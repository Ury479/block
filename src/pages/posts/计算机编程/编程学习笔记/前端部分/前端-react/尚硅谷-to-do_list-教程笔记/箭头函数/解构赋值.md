## 两种写法的本质区别

### 1. `const { xxx } = obj` — 解构赋值（从对象中**取出**值）

```js
const todo = { id: '001', name: 'eat', done: true }

// 从 todo 中取出 name 和 done
const { name, done } = todo

// 等价于：
const name = todo.name   // 'eat'
const done = todo.done   // true
```

你项目中的用法：
```jsx
// App.jsx — 从 state 中取出 todos
const { todos } = this.state
// 等价于 const todos = this.state.todos

// Item/index.jsx — 从 props 中取出 id, name, done
const { id, name, done } = this.props
// 等价于：
// const id = this.props.id
// const name = this.props.name
// const done = this.props.done
```

### 2. `const xxx = { ... }` — 对象创建（**组装**一个新对象）

```js
// 创建一个新对象
const todo = { id: '001', name: 'eat', done: true }

// App.jsx 中的 state 就是这样创建的
state = {
  todos: [
    { id: '001', name: 'eat', done: true },
    { id: '002', name: 'fuck', done: true },
  ]
}
```

## 对比总结

| | `const { x } = obj` | `const obj = { x }` |
|---|---|---|
| 方向 | **拆**：从对象中提取 | **装**：创建新对象 |
| 功能 | 取值的快捷方式 | 存值的方式 |
| `{}` 含义 | 解构模式（匹配键名） | 对象字面量（定义键值对） |
| 类比 | 拆快递，按名字取包裹 | 打包快递，放入物品 |
