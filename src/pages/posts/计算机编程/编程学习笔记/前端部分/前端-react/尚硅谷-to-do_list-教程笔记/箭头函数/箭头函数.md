### 1. 共通结构（每个组件都有的模式）

```jsx
// 所有组件都遵循这个骨架
import React, { Component } from 'react'   // ① 导入
import './index.css'                        // ② 样式

export default class Xxx extends Component { // ③ 类组件
    state = { ... }                          // ④ 自身状态（可选）
    
    handleXxx = () => { ... }                // ⑤ 事件方法（箭头函数）
    
    render() {                               // ⑥ 渲染
        const {...} = this.props             // ⑦ 解构 props
        const {...} = this.state             // ⑧ 解构 state
        return ( <div>...</div> )            // ⑨ 返回 JSX
    }
}
```

### 2. State vs Props 的分工

```
┌─────────────────────────────────────────────┐
│  State = 自己的数据（可以改）                  │
│  Props = 别人给的数据/方法（只读）              │
└─────────────────────────────────────────────┘
```

| | state | props |
|---|---|---|
| 谁定义 | 自己 | 父组件传入 |
| 能改吗 | ✅ `this.setState()` | ❌ 只读 |
| 用途 | 组件内部状态 | 父子通信 |

**在你的项目中：**

```
App 的 state:
    todos: [{id, name, done}, ...]     ← 唯一数据源！

Item 的 state:
    mouse: true/false                  ← 仅控制自身鼠标悬停样式
```

**只有 App 有 `todos` 数据，其他组件全靠 props 拿。**

---

### 3. 传参逻辑（Props 链路）

```
App (数据中心)
 │
 │  state: { todos: [...] }
 │  方法: addTodo, updateTodo, deleteTodo
 │
 ├── Header
 │     props: { addTodo }
 │     作用: 输入框 → 创建新 todo → 调 addTodo(todoObj)
 │                                      ↑ 数据往上传
 │
 ├── List
 │     props: { todos, updateTodo, deleteTodo }
 │     作用: 中间人，全部透传给 Item
 │     │
 │     └── Item (×N 个)
 │           props: { id, name, done, updateTodo, deleteTodo }
 │           作用: 显示单条 + 勾选/删除 → 调父组件方法
 │
 └── Footer
       props: { }  (暂无)
```

**关键理解：数据向下流，操作向上传**

```
数据（todos）:  App → List → Item        向下流（props）
操作（改数据）:  Item → List → App        向上传（调 props 里的函数）
```

### 4. 箭头函数 `() => {}` 的三种用法

#### 用法 A：定义方法（绑定 this）

```jsx
// 如果用普通函数：
addTodo(todoObj) {
    this.setState(...)  // ❌ this 是 undefined！
}

// 用箭头函数：
addTodo = (todoObj) => {
    this.setState(...)  // ✅ this 指向组件实例
}
```

**为什么？** 箭头函数没有自己的 `this`，它用外层的 `this`（即组件实例）。

#### 用法 B：高阶函数（返回函数）

```jsx
handleMouse = (flag) => {       // 第一层：接收参数 flag
    return () => {              // 第二层：返回真正的事件处理函数
        this.setState({mouse: flag})
    }
}

// 调用时：
onMouseEnter={this.handleMouse(true)}
//            ↑ 立即执行第一层，返回第二层
//              React 拿到的是 () => { setState({mouse: true}) }
//              鼠标进入时才执行第二层
```

**执行时间线：**
```
组件渲染时：handleMouse(true) 执行 → 返回 () => setState(...)
鼠标移入时：() => setState({mouse: true}) 执行
```

#### 用法 C：行内箭头函数（延迟调用）

```jsx
onClick={() => this.hanleDelete(id)}
//       ↑ 包一层箭头函数，只在点击时才执行 hanleDelete
```

**对比错误写法：**
```jsx
onClick={this.hanleDelete(id)}   // ❌ 渲染时立即执行！
onClick={() => this.hanleDelete(id)}  // ✅ 点击时才执行
```

### 5. 三个方法的对比（App 中的共通模式）

```jsx
// 全部遵循：取 → 改 → 存

addTodo = (todoObj) => {
    const {todos} = this.state          // 取
    const newTodos = [todoObj,...todos]  // 改（加到前面）
    this.setState({todos:newTodos})     // 存
}

updateTodo = (id,done) => {
    const {todos} = this.state          // 取
    const newTodos = todos.map(...)     // 改（改 done 值）
    this.setState({todos:newTodos})     // 存
}

deleteTodo = (id) => {
    const {todos} = this.state          // 取
    const newTodos = todos.filter(...)  // 改（过滤掉）
    this.setState({todos:newTodos})     // 存
}
```

| 操作 | 数组方法 | 作用 |
|---|---|---|
| 增 | `[new, ...old]` | 展开旧数组，新项放前面 |
| 改 | `.map()` | 遍历，匹配的改，不匹配的原样返回 |
| 删 | `.filter()` | 遍历，不匹配的保留，匹配的丢弃 |

**都不直接修改原数组**，而是创建新数组 → 这是 React 的原则：**不可变数据**。

![Pasted image 20260203113816.png](/media/3c763205d2a03468.png)
- xxx = () => {}：在类组件里最常用的意义是 **“不用 bind，this 不会丢”**
- 伪代码理解：**渲染时 ref 把 DOM 存到 this.refs；事件触发时从 this.refs 拿 DOM，再读 value。**
它的核心作用有 3 个：
### **作用 A：把方法变成“实例自己的属性”（不是原型方法）**

- showData = ... 会把函数放到 **每个组件实例(this)** 上
    
- 而 showData() { ... }（不带 =）是放到 **原型 prototype** 上
    
### **作用 B：自动固定** 

### **this**

###  **指向（最关键）**
箭头函数有一个重要特性：**它没有自己的 this**，它会使用外层的 this（这里外层就是组件实例）。

所以：
- showData = () => { ... } 里 this 一直是组件实例
- 你可以直接写：onClick={this.showData}，不用再 .bind(this)

### **一、为什么写成** 

## **add = () => {}**

##  **？**

  

这是：

  

> 类字段 + 箭头函数写法

  

作用有两个：

  

### **① 自动绑定** 

### **this**

  

在普通类方法中：

```
add() {
  console.log(this);
}
```

如果直接传给事件：

```
<button onClick={this.add}>点击</button>
```

此时 this 会丢失（变成 undefined）。

  

必须手动绑定：

```
constructor() {
  super();
  this.add = this.add.bind(this);
}
```

---

### **② 使用箭头函数可以避免 bind**

  

箭头函数没有自己的 this，

它会继承外层作用域的 this。

```
add = () => {
  console.log(this);
}
```

这里的 this 永远指向组件实例。

  

所以你不需要再写：

```
this.add = this.add.bind(this);
```

---

### **二、对比三种写法**

  

### **❌ 旧写法（容易丢失 this）**

```
add() {
  console.log(this);
}
```

---

### **⚠ 旧解决方案**

```
constructor() {
  super();
  this.add = this.add.bind(this);
}
```

---

### **✅ 推荐写法（你现在看到的）**

```
add = () => {
  console.log(this);
}
```

简洁、安全、现代写法。

---

### **三、你说的“简化表达式”不完全准确**

  

它不是“简化表达式”，

  

它的核心目的：

  

> 保证 this 永远指向当前组件实例。

---

### **四、为什么 React 项目中大量使用箭头函数？**

  

因为在 React 中：

- 事件处理函数经常需要访问 this.state
    
- 如果 this 丢失会报错
    

  

所以箭头函数成为标准写法。

---

### **五、一句话总结**

  

add = () => {} 的作用是：

  

> 使用箭头函数自动绑定 this，避免手动 bind。

## **1）事件触发模块：按钮点击 onClick**

  

### **代码**

```
<button onClick={() => removeTask(t.id)}>Delete</button>
```

### **结构拆解**

- onClick={...}：React 事件绑定（不是字符串，是函数）
    
- () => removeTask(t.id)：**箭头函数包一层**，目的：点击时才执行
    
- t.id：把当前列表项 id 传给删除函数
    

  

### **你必须懂的前置**

- 箭头函数 () => {}
    
- 函数调用与“传函数”区别
    
    - onClick={removeTask(t.id)} ❌（会立刻执行）
        
    - onClick={() => removeTask(t.id)} ✅（点击才执行）
        
    

---

## **2）列表渲染模块：array.map → JSX 列表**

  

### **代码**

```
cart.map((x) => (
  <div key={x.id}>
    ...
  </div>
))
```

### **结构拆解**

- cart.map(...)：把数组每个元素转换成一个 JSX 节点
    
- (x) => (...)：回调函数返回 JSX
    
- key={x.id}：React 用 key 识别每一项（用于更新/重排）
    

  

### **你必须懂的前置**

- Array.prototype.map
    
- React 列表必须有 key（且 key 应稳定唯一）
    

---

## **3）组件展示模块：JSX 中嵌入表达式** 

## **{}**

  

### **代码**

```
${x.price} × {x.qty}
```

### **结构拆解**

- JSX 里 **只有表达式** 能放进 {}，不能放 if/for
    
- 字符串 $ 是纯文本；{x.price} 是 JS 表达式
    

  

### **前置**

- JSX 规则：{} 内只能写表达式
    
    ✅ a ? b : c / arr.map / a && b
    
    ❌ if(...) {} / for(...) {}

## **4）操作函数模块：删除（filter）**

  

### **典型结构（你评论系统里就有）**

```
function deleteComment(id) {
  setComments((prev) => prev.filter((c) => c.id !== id));
}
```

### **结构拆解**

- setComments((prev) => ...)：使用函数式更新，拿到最新 prev
    
- filter：返回一个新数组（不可变更新）
    
- c.id !== id：保留不是目标 id 的项
    

  

### **前置**

- filter 返回新数组
    
- 不可变更新（不要 prev.splice）
    

---

## **5）操作函数模块：修改某一项（map + 条件 + 展开运算符）**

  

### **代码（点赞）**

```
function likeComment(id) {
  setComments((prev) =>
    prev.map((c) => (c.id === id ? { ...c, likes: c.likes + 1 } : c))
  );
}
```

### **结构拆解（这是最核心的一块）**

- prev.map(...)：遍历每个元素，返回新数组
    
- c.id === id ? ... : c：只改命中项，其它保持原样
    
- { ...c, likes: c.likes + 1 }：复制旧对象 + 覆盖 likes（不可变更新）
    

  

### **前置**

- 三元运算符 cond ? A : B
    
- 对象展开 {...obj}（ES6）
    
- “对象不可变更新”的习惯
    

---

## **6）新增模块：头插入（[new, …prev]）**

  

### **代码（任务/评论一样）**

```
setTasks((prev) => [
  { id: String(Date.now()), title: value, done: false, priority },
  ...prev,
]);
```

### **结构拆解**

- [...]：创建新数组
    
- {...}：新任务对象
    
- ...prev：把旧数组展开到新数组后面（形成头插）
    
- String(Date.now())：生成简单 id（真实项目建议 uuid）
    

  

### **前置**

- 数组展开 [...arr]
    
- 对象字面量 {a:1}
    
- ES6 属性简写：priority 等价于 priority: priority
    

---

## **7）表单提交模块：e.preventDefault + 受控输入**

  

### **代码**

```
function addTask(e) {
  e.preventDefault();
  const value = title.trim();
  if (!value) return;
  ...
}
```

### **结构拆解**

- e.preventDefault()：阻止表单默认刷新页面
    
- title.trim()：去掉空格
    
- if (!value) return;：输入校验（空就不新增）
    

  

### **受控输入对应代码**

```
<input value={text} onChange={(e) => setText(e.target.value)} />
```

### **前置**

- DOM 事件对象 e
    
- 字符串 trim()
    
- 受控组件：value 永远来自 state

## **8）组件模块结构：App / CommentDemo / KeyPatterns**

  

### **代码片段**

```
export default function App() {
  return <CommentDemo />;
}

function CommentDemo() { ... }

function KeyPatterns({ items }) { ... }
```

### **结构拆解**

- App：入口组件（根）
    
- CommentDemo：业务组件（真正逻辑在这里）
    
- KeyPatterns：纯展示组件（只负责渲染 props）
    

  

### **重要区分（你必须掌握）**

- **容器组件**：负责 state / 逻辑 / 请求（CommentDemo）
    
- **展示组件**：只接收 props 渲染（KeyPatterns）

# **归纳总结：你贴的代码只用了 3 个“数组套路”**

  

把它们背下来，所有列表项目都通了：

  

## **A. 新增（头插）**

```
setX(prev => [newItem, ...prev])
```

## **B. 删除（filter）**

```
setX(prev => prev.filter(i => i.id !== id))
```

## **C. 修改（map + 条件 + 展开）**

```
setX(prev => prev.map(i => i.id === id ? { ...i, field: newValue } : i))
```
