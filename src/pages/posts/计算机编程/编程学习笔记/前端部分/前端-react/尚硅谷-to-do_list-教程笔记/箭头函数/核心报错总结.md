# 函数，state状态，闭包 部分
## 一、箭头函数

### 你犯过的错

```jsx
// 错误 1：函数体 vs 简写混淆
todos.map((todo) => 
    return {<Item/>    // ❌ 简写模式不能写 return
})

// 错误 2：空参数
addTodo = () => { ... }  // ❌ 谁传数据进来？
```

### 核心规则

```jsx
// 简写：省略 {} 和 return，直接返回表达式
todos.map((todo) => <Item key={todo.id} {...todo}/>)

// 完整：用 {} 包裹，必须写 return
todos.map((todo) => {
    return <Item key={todo.id} {...todo}/>
})

// 返回对象字面量：必须加 () 消除歧义
const makeTodo = () => ({ id: '001', name: 'eat' })
```

### 判断方法

```
=> 后面跟什么？
  跟 { }  → 函数体（需要 return）
  跟表达式 → 简写（自动返回）
  跟 ({ }) → 返回对象
```

## 二、高阶函数

### 你犯过的错

```jsx
// 错误：不理解为什么要返回函数
handleMouse = (flag) => {
    this.setState({mouse: flag})  // ❌ 直接执行，渲染时就触发了
}
```

### 核心规则

```jsx
// 高阶函数 = 返回函数的函数
handleMouse = (flag) => {       // 外层：渲染时调用，绑定参数
    return () => {              // 内层：事件触发时调用
        this.setState({ mouse: flag })
    }
}

handleCheck = (id) => {         // 外层：渲染时调用，绑定 id
    return (event) => {         // 内层：点击时调用，React 传入 event
        this.props.updateTodo(id, event.target.checked)
    }
}
```

### 两层参数的时机

```
┌────────────────────────────────────────┐
│ 渲染时（你提前知道的）                    │
│   handleMouse(true)    → flag = true    │
│   handleCheck('002')   → id = '002'     │
├────────────────────────────────────────┤
│ 事件触发时（用户操作后才知道的）           │
│   () => { ... }        → 无参数          │
│   (event) => { ... }   → React 传入     │
└────────────────────────────────────────┘
```

## 三、state

### 你犯过的错

```jsx
// 错误 1：没有定义 state 就使用
this.state.mouse  // ❌ state 没定义，undefined

// 错误 2：从 state 取不存在的键
const { id } = this.state  // ❌ state 里没有 id，只有 todos

// 错误 3：setState 传入非对象
this.setState(flag)  // ❌ 布尔值不是对象

// 错误 4：setState 传入函数
this.setState({ todos: this.updateTodo })  // ❌ 函数不是数据
```

### 核心规则

```jsx
// 1. 先定义
state = { todos: [...], mouse: false }

// 2. 取值（解构）
const { todos } = this.state   // ✅ 键名必须匹配

// 3. 更新（必须传对象）
this.setState({ todos: newTodos })   // ✅ { 键: 新值 }
this.setState({ mouse: true })       // ✅
this.setState(true)                  // ❌ 不是对象
this.setState(this.updateTodo)       // ❌ 不是数据

// 4. 不可变更新：永远创建新的，不直接修改原数据
const newTodos = [todoObj, ...todos]              // 新数组
const newTodos = todos.map(t => ({ ...t, done })) // 新数组 + 新对象
this.setState({ todos: newTodos })
```

## 四、props

### 你犯过的错

```jsx
// 错误 1：props 当函数调用
this.event.props({})  // ❌ props 是对象，不是函数

// 错误 2：this.event（不存在）
this.event.target.checked  // ❌ event 是函数参数，不在 this 上

// 错误 3：没传 updateTodo 给 List
<List todos={todos}/>  // ❌ 缺少 updateTodo，Item 无法调用
```

### 核心规则

```jsx
// props 是父组件传给子组件的数据/方法

// App（父）传递
<Header addTodo={this.addTodo}/>
<List todos={todos} updateTodo={this.updateTodo}/>

// List（中转）继续传
<Item key={todo.id} {...todo} updateTodo={updateTodo}/>

// Item（子）使用
const { id, name, done } = this.props   // 取数据
this.props.updateTodo(id, checked)       // 调方法
```

### 传递链路

```
App.state（数据源）
  │
  ├─→ Header.props.addTodo        （App 的方法）
  │
  ├─→ List.props.todos            （App 的数据）
  │   List.props.updateTodo        （App 的方法）
  │     │
  │     └─→ Item.props.id/name/done  （单条数据）
  │         Item.props.updateTodo     （App 的方法）
  │
  └─→ Footer.props                 （待实现）
```

## 五、闭包

### 你犯过的错

```jsx
// 不理解内层函数为什么能访问外层的 id
handleCheck = (id) => {
    return (event) => {
        // id 从哪来？→ 闭包：内层函数记住了外层的 id
        this.props.updateTodo(id, event.target.checked)
    }
}
```

### 核心规则

```jsx
// 闭包 = 内层函数"记住"外层函数的变量

// 例 1：handleMouse
handleMouse = (flag) => {        // flag 在这里
    return () => {
        this.setState({ mouse: flag })  // 这里还能用 flag → 闭包
    }
}

// 例 2：handleCheck
handleCheck = (id) => {          // id 在这里
    return (event) => {
        updateTodo(id, ...)      // 这里还能用 id → 闭包
    }
}

// 例 3：updateTodo 里的 map
updateTodo = (id, done) => {     // id, done 在这里
    todos.map((todoObj) => {
        if (todoObj.id === id)   // 这里还能用 id → 闭包
        return { ...todoObj, done }  // 这里还能用 done → 闭包
    })
}
```

## 六、解构 & 展开运算符

### 你犯过的错

```jsx
// 错误 1：解构名与 state 键名不匹配
const { id } = this.state    // ❌ state 里没有 id
const { todos } = this.state // ✅

// 错误 2：[] 和 {} 混用
return [...todoObj, done]   // ❌ 展开对象放进数组，结构全乱
return { ...todoObj, done } // ✅ 展开对象，覆盖 done 属性

// 错误 3：解构了不用
const { mouse } = this.state
this.state.mouse  // ❓ 那解构的意义呢？直接用 mouse
```

### 核心规则

```
= 左边 {} → 解构（拆）  const { name } = this.props
= 右边 {} → 创建（装）  const obj = { name: 'eat' }

[] 展开数组：[todoObj, ...todos]     → 新数组
{} 展开对象：{ ...todoObj, done }    → 新对象（后面覆盖前面）
```

## 速查表：遇到报错先检查

| 检查项             | 问自己                                    |
| --------------- | -------------------------------------- |
| state 定义了吗？     | `state = { ... }` 写了没？                 |
| 解构名对吗？          | state 里的键名和解构变量名一致吗？                   |
| setState 传对象了吗？ | `setState({ 键: 值 })`，不是函数、不是布尔值        |
| props 传完整了吗？    | 子组件需要的方法，父组件传了吗？中间层转发了吗？               |
| 高阶函数返回函数了吗？     | 需要传参的事件处理器，用了 `return () => {}` 吗？     |
| `[]` 还是 `{}`？   | 操作数组用 `[]`，操作对象用 `{}`                  |
| event 直接用了吗？    | `event.target.checked`，不是 `this.event` |
|                 |                                        |

# 箭头函数，解构赋值，Hooks,props 部分
根据整个对话中你出现的所有错误，总结如下：

---

## 知识盲区一：解构赋值 vs 函数定义 — 混淆最严重

```jsx
// 你写的（错）：把两种语法混在一起
const { handleAdd } = (payload) => { addTodo(payload) }

// 解构赋值 —— 从对象/数组取值，右边必须是对象
const { todos, addTodo } = useTodos()

// 定义函数 —— 左边是变量名，不加 {}
const handleAdd = (payload) => { addTodo(payload) }
```

**核心混淆：** `const { xxx }` 和 `const xxx` 是完全不同的语法，不能根据感觉混用。

---

## 知识盲区二：调用对象搞错 — Hook vs 组件

```jsx
// 你写的（错）
const { todoList } = TodoListPage()   // 自己调用自己

// 正确
const { todos } = useTodos()          // 调用 Hook
```

**核心混淆：** 不清楚**数据从哪里来**。数据和方法来自 `useTodos` 这个 Hook，不是来自页面组件自身。

---

## 知识盲区三：`props` 的使用范围

```jsx
// 你写的（错）
props.toggleTodo(...)    // TodoListPage 里没有 props

// 正确：方法从 useTodos 解构，直接用
todos.forEach(...)
```

**核心混淆：** `props` 只在**子组件接收父组件数据**时存在。`TodoListPage` 是顶层页面组件，直接用 Hook，不需要 `props`。

---

## 知识盲区四：`forEach` 用在了函数上

```jsx
// 你写的（错）
handleAdd.forEach(...)   // handleAdd 是函数，函数没有 .forEach

// 正确：.forEach 只能用在数组上
todos.forEach(...)
```

**核心混淆：** 不清楚 **`.forEach()` 是数组的方法**，任何非数组的东西调用它都会报错。

---

## 知识盲区五：JSX 闭合标签语法

```jsx
// 你写的（错）：自闭合，不是闭合标签
<Link to="/todos">Todo 列表<Link/>

// 正确：闭合标签斜杠在标签名前面
<Link to="/todos">Todo 列表</Link>
```

**核心混淆：** `<Link/>` 是自闭合（新开一个标签），`</Link>` 才是关闭前面的标签。

---

## 知识盲区六：嵌套路由结构

```jsx
// 你写的（错）：平级，Outlet 永远不渲染内容
<Route path="/" element={<RootLayout/>}/>
<Route path="todos" element={<TodoListPage />}/>

// 正确：嵌套，子页面渲染在 Outlet 里
<Route path="/" element={<RootLayout/>}>
  <Route path="todos" element={<TodoListPage />}/>
</Route>
```

**核心混淆：** 不了解 `<Outlet />` 的工作原理 — 它只渲染**嵌套在自身内部**的子路由。

---

## 总结

| 盲区 | 本质问题 |
|------|---------|
| 解构 vs 函数定义 | JS 基础语法不熟 |
| Hook vs 组件 | 不清楚数据来源 |
| props 范围 | 不清楚组件通信方向 |
| forEach 用在函数上 | 不清楚数组方法归属 |
| JSX 闭合标签 | JSX 语法不熟 |
| 嵌套路由 | 不理解 Outlet 机制 |

**优先巩固：** JS 解构赋值、数组方法（`map/filter/forEach`）、React 数据流（Hook → 组件 → props）这三块是基础，搞清楚后其他问题会自然减少。
