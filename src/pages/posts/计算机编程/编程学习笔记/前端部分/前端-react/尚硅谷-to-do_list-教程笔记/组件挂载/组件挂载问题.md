## `this.state.mouse` 的来源

### 第一步：初始化（你的代码**缺了这一步**）

```jsx
export default class Item extends Component {
    // ⭐ state 在这里定义，mouse 的初始值是 false
    state = { mouse: false }
    //        ↑
    //        this.state.mouse 就是从这里来的
}
```

### 第二步：更新

```jsx
handleMouse = (flag) => {
    return () => {
        this.setState({ mouse: flag })
        //                      ↑
        //   flag 是 true 或 false，写入 state.mouse
        //   更新后 this.state.mouse 就变成了 flag 的值
    }
}
```

### 第三步：读取

```jsx
<li style={{ backgroundColor: this.state.mouse ? '#ddd' : '#fff' }}>
//                             ↑
//                             从 state 中读取 mouse 的当前值
```

## 完整流程

```
① 组件挂载
   state = { mouse: false }       ← this.state.mouse = false

② 鼠标移入，触发 handleMouse(true)
   this.setState({ mouse: true }) ← this.state.mouse = true
                          ↑
                        flag = true

③ 组件重新渲染
   this.state.mouse === true → 背景色 '#ddd'

④ 鼠标移出，触发 handleMouse(false)
   this.setState({ mouse: false }) ← this.state.mouse = false
                          ↑
                        flag = false

⑤ 组件重新渲染
   this.state.mouse === false → 背景色 '#fff'
```

## 回答你的问题

> 是通过 flag 传入的参数吗？

**不完全是。** `this.state.mouse` 来自**两个地方**：

| 阶段 | 来源 | 值 |
|------|------|-----|
| 初始 | `state = { mouse: false }` | `false` |
| 更新 | `this.setState({ mouse: flag })` | `flag` 的值（`true` 或 `false`） |

`flag` 只是更新时的**传递工具**，`state` 才是**存储位置**。`this.state.mouse` 读的永远是 `state` 里存的值。
