## 你忽略的部分

对比截图中教程的 Header 和你当前的代码：

```jsx
// 教程 Header ✅（截图中的完整逻辑）
handleKeyUp = (event) => {
    const {keyCode, target} = event
    if(keyCode !== 13) return
    console.log(target.value)               // ← 你缺
    this.props.addTodo(target.value)         // ← 你缺
}

// 你的 Header ❌（按回车后什么都没做）
handleKeyUp = (event) => {
    const {keyCode, target} = event
    if(keyCode !== 13) return
    // 到这里就结束了，回车之后... 然后呢？
}
```

**你漏掉了按回车后的两步关键操作：**

### 缺失 1：没有调用 `this.props.addTodo()`

App 已经把 `addTodo` 通过 props 传给了 Header：
```jsx
// App.jsx 第29行
<Header addTodo={this.addTodo}/>
```

但 Header 里从没调用过它。就像快递员（App）给你留了电话号码（`addTodo`），你需要打代码（Header 按回车），结果你忘了拨出去。

### 缺失 2：没有把输入的值传出去

`target.value` 就是用户在输入框里打的文字，这个值需要传给 `addTodo` 才能创建新的 todo。

---

## 完整的调用链（你需要补的环节打了 ⭐）

```
用户在输入框打字 → 按回车
    ↓
Header.handleKeyUp 触发
    ↓
keyCode === 13？不是就 return
    ↓
⭐ this.props.addTodo(target.value)  ← 把输入值传给 App
    ↓
App.addTodo(todoObj) 执行
    ↓
创建新数组 → setState → 页面更新
```

不过注意：截图中教程传的是 `target.value`（字符串），但你 App 里的 `addTodo` 期望接收的是一个对象 `todoObj = {id, name, done}`。所以 Header 里还需要先**组装成对象**再传：

```jsx
handleKeyUp = (event) => {
    const {keyCode, target} = event
    if(keyCode !== 13) return
    // 组装 todo 对象
    const todoObj = { id: Date.now().toString(), name: target.value, done: false }
    // 传给 App
    this.props.addTodo(todoObj)
    // 清空输入框
    target.value = ''
}
```

教程截图中直接传字符串可能是简化写法，后续会在 App 的 `addTodo` 里组装对象。你的 App 已经期望接收对象了，所以在 Header 里组装好再传更完整。
