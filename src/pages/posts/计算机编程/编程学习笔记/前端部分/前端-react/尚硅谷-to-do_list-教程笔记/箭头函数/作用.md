`{}`涉及数据传递，包括储存数据(键值对)和调用数据(函数)

![Pasted image 20260227114650.png](/media/10545b96308c0ed1.png)

## `{}` 在 JS 中的两种含义

```js
// 1. 对象字面量 — 用来创建数据（一组键值对）
const todo = { id: '001', name: 'eat', done: true }

// 2. 函数体 — 用来包裹要执行的代码块
const fn = () => { return <Item/> }
```

## 核心区别

| | 对象字面量 `{}` | 函数体 `{}` |
|---|---|---|
| 本质 | **数据结构**，描述"是什么" | **代码块**，描述"做什么" |
| 内容 | 键值对 `{ key: value }` | 语句 `{ statement; return ... }` |
| 类比 | 一张配置表 / props | 一个函数 / 组件的 render 方法 |

## 你的报错就是这个歧义

```jsx
// 箭头函数后面紧跟 {} — JS 引擎认为是「函数体」
todos.map((todo) => { return <Item/> })  // ✅ 函数体

// 但你写的：
todos.map((todo) => 
  return {<Item/>    // ❌ JS 看到 => 后没有 {，认为是简写模式
})                   //    简写模式不能写 return
```

## 箭头函数的两种写法对比

```jsx
// 写法 A：简写（省略 {} 和 return，直接返回表达式）
todos.map((todo) => <Item key={todo.id} {...todo}/>)

// 写法 B：完整（用 {} 包裹函数体，必须写 return）
todos.map((todo) => {
  return <Item key={todo.id} {...todo}/>
})

// 如果简写要返回对象字面量，必须加 () 消除歧义
const makeTodo = () => ({ id: '001', name: 'eat' })
//                       ^                       ^  告诉 JS：这是对象，不是函数体
```

## 回到你的类比

> 对象字面量是参数，函数体是组件

更准确的说法：
- **对象字面量** → 类似 React 的 **props / state**（存放数据）
- **函数体** → 类似 React 的 **render 方法**（执行逻辑，产出结果）

```jsx
// state 就是对象字面量
state = { todos: [{id:'001', name:'eat', done:true}] }

// render 就是函数体
render() {
  return <div>...</div>
}
```

两者配合：**对象存数据，函数用数据**。
