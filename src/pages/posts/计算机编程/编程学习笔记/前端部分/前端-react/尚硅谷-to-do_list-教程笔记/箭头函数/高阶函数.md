## `handleMouse` 的作用

它不是"将数据存入 flag"，而是一个**高阶函数**（返回函数的函数），用来控制鼠标悬停时的背景色和删除按钮显示。

```jsx
// flag 不是状态，是参数（true 或 false）
handleMouse = (flag) => {
    // 返回一个新函数，这个新函数才是真正的事件处理器
    return () => {
        this.setState({ mouse: flag })  // 把 flag 的值写入 state.mouse
    }
}
```

## 执行流程

```
鼠标移入 li
  ↓
onMouseEnter={this.handleMouse(true)}
  ↓
handleMouse(true) 被调用，flag = true
  ↓
返回 () => { this.setState({mouse: true}) }
  ↓
React 执行这个返回的函数
  ↓
state.mouse 变为 true → 触发重新渲染

鼠标移出 li
  ↓
onMouseLeave={this.handleMouse(false)}
  ↓
同理，state.mouse 变为 false
```

## 各角色的职责

| 名称 | 类型 | 作用 |
|------|------|------|
| `flag` | **参数** | 传入 true/false，决定鼠标是进入还是离开 |
| `state.mouse` | **状态** | 记录当前鼠标是否在此 li 上 |
| `handleMouse` | **高阶函数** | 根据 flag 生成不同的事件处理函数 |

## 为什么要用高阶函数？

因为 `onMouseEnter` 需要的是一个**函数引用**，而不是一个函数调用的结果：

```jsx
// ❌ 直接调用会立刻执行 setState，而不是等鼠标事件
onMouseEnter={this.setState({mouse: true})}

// ❌ 想传参但箭头函数直接写在 JSX 里也行，但不够优雅
onMouseEnter={() => this.setState({mouse: true})}

// ✅ 高阶函数：handleMouse(true) 返回一个函数，交给 React 在事件触发时调用
onMouseEnter={this.handleMouse(true)}
```

**一句话总结**：`flag` 是参数不是状态，`handleMouse` 是一个"生成事件处理器的工厂"，`state.mouse` 才是真正的状态，控制 li 的背景色和删除按钮是否显示。
