#### **2. Event Loop Execution Order**

1. **What will be the output of the following code?**
    
    ```javascript
    console.log("A");
    setTimeout(() => console.log("B"), 100);
    setTimeout(() => console.log("C"), 0);
    console.log("D");
    ```
    
    - **Your Answer:** b ✅ (Correct)
    - **Explanation:**
        - **Step 1:** `"A"` is logged immediately (synchronous code).
        - **Step 2:** The first `setTimeout(() => console.log("B"), 100)` schedules `"B"` to run **after 100ms**.
        - **Step 3:** The second `setTimeout(() => console.log("C"), 0)` schedules `"C"` to run **as soon as the call stack is clear**.
        - **Step 4:** `"D"` is logged immediately (synchronous code).
        - **Step 5:** The event loop picks up `"C"` (since it's the first scheduled task with `0ms` delay).
        - **Step 6:** After 100ms, `"B"` is finally logged.
        - **Final Output:** **A → D → C → B**

### **5. How is `setImmediate()` different from `setTimeout(fn, 0)`?**

📌 **Challenge:** Many assume `setTimeout(fn, 0)` and `setImmediate(fn)` are equivalent. Explain the difference and when to use each.

- **Your Answer:**  
    _"`setTimeout(fn, 0)` will hold for a period of time to execute while `setImmediate(fn)` will execute the function immediately."_ ❌
- **Correction:**
    - **`setTimeout(fn, 0)`** schedules a function to run **after at least 0ms**, but in reality, it depends on the event loop and may be slightly delayed.
    - **`setImmediate(fn)`** executes the function **after the current I/O operations complete** but **before the next tick of the event loop**.
    - **Order of Execution:**
        - If both are executed inside the main script, **`setImmediate()` runs first**.
        - If executed inside an I/O callback, **`setTimeout(fn, 0)` runs first**.
- **Corrected Answer:**  
    _"`setTimeout(fn, 0)` schedules execution in the **next event loop cycle** but may be delayed due to the event queue. `setImmediate(fn)` runs **immediately after I/O callbacks** but before the next event loop tick. In most cases, `setImmediate()` executes first, unless inside an I/O operation."_

1. **Create a simple Node.js server using Express.js that responds with "Welcome to my Node.js Server" when the root URL is accessed.**  
    **Expected Output:**
    
    - Visiting `http://localhost:3000/` in a browser should display `"Welcome to my Node.js Server"`.  
        **Solution:**
    
    ```javascript
    const express = require("express");
    const app = express();
    
    app.get("/", (req, res) => {
        res.send("Welcome to my Node.js Server");
    });
    
    app.listen(3000, () => {
        console.log("Server running on port 3000");
    });
    ```
    
2. **Modify the above server to include a `/contact` route that returns the message "Contact me at [email@example.com](mailto:email@example.com)".**  
    **Expected Output:**
    
    - Visiting `http://localhost:3000/contact` in a browser should display `"Contact me at email@example.com"`.  
        **Solution:**
    
    ```javascript
    app.get("/contact", (req, res) => {
        res.send("Contact me at email@example.com");
    });
    ```
    
3. **Write an Express server that accepts two numbers via a POST request and returns their sum.**  
    **Expected Output:**
    
    - Sending a POST request with `num1=5` and `num2=10` should return `"The sum is 15"`.  
        **Solution:**
    
    ```javascript
    const express = require("express");
    const bodyParser = require("body-parser");
    
    const app = express();
    app.use(bodyParser.urlencoded({ extended: true }));
    
    app.post("/", (req, res) => {
        const sum = Number(req.body.num1) + Number(req.body.num2);
        res.send("The sum is " + sum);
    });
    
    app.listen(3000, () => {
        console.log("Server running on port 3000");
    });
    ```
