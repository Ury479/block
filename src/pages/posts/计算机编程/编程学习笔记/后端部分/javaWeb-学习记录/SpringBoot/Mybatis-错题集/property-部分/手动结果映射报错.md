图中内容主要涉及 MyBatis 中的数据封装技术，具体包括起别名、手动结果映射和开启驼峰命名规则。这些技术用于在 Java 应用中处理数据库查询结果时，确保数据库字段和 Java 对象属性之间的映射正确无误。以下是对每个部分的详细解释：

### 1. 起别名
- **目的**：在 SQL 查询中为字段起别名，使得数据库字段名与 Java 实体类的属性名一致。
- **示例**：
  ```java
  @Select("select id, username, password, name, gender, image, job, entrydate, dept_id deptId, create_time createTime, update_time updateTime from emp where id = #{id}")
  public Emp getById(Integer id);
  ```
  - 这里 `dept_id` 被起别名为 `deptId`，`create_time` 被起别名为 `createTime`，`update_time` 被起别名为 `updateTime`，以便与 Java 类中的属性名一致。

### 2. 手动结果映射
- **目的**：通过注解 `@Results` 和 `@Result` 手动指定数据库字段与 Java 对象属性之间的映射关系。
- **示例**：
  ```java
  @Select("select * from emp where id = #{id}")
  @Results({
      @Result(column = "dept_id", property = "deptId"),
      @Result(column = "create_time", property = "createTime"),
      @Result(column = "update_time", property = "updateTime")
  })
  public Emp getById(Integer id);
  ```
  - 使用 `@Results` 和 `@Result` 明确指定数据库字段与实体类属性的映射关系。

### 3. 开启驼峰命名
- **目的**：自动将数据库字段名（下划线命名）映射为 Java 属性名（驼峰命名）。
- **配置**：
  ```properties
  mybatis.configuration.map-underscore-to-camel-case=true
  ```
  - 开启此配置后，MyBatis 会自动将数据库字段名 `a_column` 映射为 Java 属性名 `aColumn`。
