下面把 “MyBatis 驼峰映射” 讲清楚，并重点说：当**字段名和属性名完全不同**时，怎么映射。

# 一、什么是驼峰映射？

- 目的：把**下划线风格的列名**（如 `dept_id`, `create_time`）自动映射到 Java **驼峰属性**（`deptId`, `createTime`）。
    
- 开关（Spring Boot `application.properties`）：
    

```properties
mybatis.configuration.map-underscore-to-camel-case=true
```

- 生效范围：**查询结果集 -> Java 对象** 的映射（ResultSet→POJO）。  
    对于 **SQL 参数**（Insert/Update 里的 `#{xxx}`）不受这个开关影响，`#{}` 里的名字必须用 **Java 属性名**。
    

# 二、自动映射能做什么/不能做什么

✅ 能做：

- `dept_id -> deptId`
    
- `create_time -> createTime`
    
- 大小写不敏感（MySQL 列名默认不区分大小写）
    

❌ 不能做：

- **完全不同的命名**：例如列叫 `avatar`，Java 属性叫 `image`。这不属于“下划线转驼峰”，需要手工指定映射（见下一节）。
    
- 多表 `JOIN` 导致**重复列名**（比如两个表都有 `id`），需要起别名或自定义映射。
    

# 三、当列名与属性名**完全不同**时的三种做法（推荐顺序）

> 目标：列 `avatar` → 属性 `image`

### 方法 1：SQL 别名（最简单、读起来也直观）

```java
@Select("SELECT id, username, avatar AS image FROM emp WHERE id = #{id}")
Emp getById(Long id);
```

- 只在少量字段不一致时非常好用；
    
- 也适合多表 join 时消除重复列名：`a.id AS empId, b.id AS deptId`
    

### 方法 2：注解版 @Results / @Result（Mapper 注解风格）

```java
@Mapper
public interface EmpMapper {
  @Select("SELECT id, username, avatar FROM emp WHERE id = #{id}")
  @Results(id="EmpMap", value = {
      @Result(column = "avatar", property = "image")
  })
  Emp getById(Long id);

  // 复用
  @Select("SELECT id, username, avatar FROM emp")
  @ResultMap("EmpMap")
  List<Emp> list();
}
```

- 适合很多 SQL 都要用到同一份映射时（用 `@Results(id=...)` + `@ResultMap` 复用）。
    

### 方法 3：XML `<resultMap>`（XML 风格，最灵活）

```xml
<!-- resources/mapper/EmpMapper.xml -->
<mapper namespace="com.example.demo.mapper.EmpMapper">
  <resultMap id="EmpMap" type="com.example.demo.pojo.Emp">
    <id     column="id"     property="id"/>
    <result column="username" property="username"/>
    <result column="avatar"   property="image"/>
    <!-- 还可以做复杂嵌套映射 -->
    <!-- <association> / <collection> -->
  </resultMap>

  <select id="getById" resultMap="EmpMap">
    SELECT id, username, avatar FROM emp WHERE id = #{id}
  </select>
</mapper>
```

- 适合复杂场景：多表结果、嵌套对象、集合属性等；
    
- 能复用、能精细控制每一列。
    

# 四、与“自动映射等级”的关系（更稳的自动映射）

MyBatis 的自动映射有级别：`PARTIAL`（默认）/`FULL`。  
某些场景（例如嵌套、重复列别名很多）建议显式设置：

```properties
# 自动映射级别更积极，减少“未映射列”的问题
mybatis.configuration.auto-mapping-behavior=FULL

# 遇到未知列时给出警告/失败（可选，帮助排错）
mybatis.configuration.auto-mapping-unknown-column-behavior=WARNING
```

# 五、常见坑 & 小贴士

1. **只对查询结果生效**：`map-underscore-to-camel-case` 不影响 `INSERT/UPDATE` 的 `#{}` 参数名。写 SQL 时用 Java 属性名：
    
    ```sql
    INSERT INTO emp (avatar) VALUES (#{image})   -- 右边用 Java 属性名 image
    ```
    
2. **toString**：想让控制台打印对象内容（不是 `Emp@xxxx`），在实体加 `@Data` 或写 `toString()`。
    
3. **布尔属性**：数据库 `is_deleted` → Java 建议用 `deleted`（或 `isDeleted`），驼峰映射能处理，但请保持命名一致性。
    
4. **多表重复列**：必须起别名，否则会映射到错误属性或被覆盖。  
    例：`SELECT a.id AS empId, b.id AS deptId, ...`
    
5. **枚举/特殊类型**：列值到 Java 枚举/自定义类型，可配 **TypeHandler**：
    
    ```java
    @Result(column="gender", property="gender", typeHandler=GenderEnumTypeHandler.class)
    ```
    
6. **全表 `SELECT *`**：学习阶段 OK；生产建议**显式列名**（可配别名），可读性和演进性更好。
    

# 六、把它用到你的项目（最小改动方案）

- 在 `application.properties`：
    
    ```properties
    mybatis.configuration.map-underscore-to-camel-case=true
    mybatis.configuration.log-impl=org.apache.ibatis.logging.stdout.StdOutImpl
    ```
    
- 字段完全不同时（如 `avatar`→`image`）：  
    **优先用 SQL 别名**；若多个地方复用，换成 **@Results/@Result** 或 **XML resultMap**。
