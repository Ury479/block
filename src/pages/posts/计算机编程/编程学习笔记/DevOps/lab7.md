1. What is DevOps?
	a. DevOps includes CI/CD which  is to solve the problems of deployment by automate release for software and update the version and integrate the function
	
	- **CI/CD** is about to automate test and deploy the software and ensure the performance of system and check the issue resolution

	#### **Key Features of DevOps:**
	
	- **CI/CD (Continuous Integration & Continuous Deployment)** – Automates testing and deployment.
	- **Infrastructure as Code (IaC)** – Automates server provisioning (e.g., Terraform, Ansible).
	- **Monitoring & Feedback** – Ensures system performance and quick issue resolution.

2. DevOps process flow (CI/CD pipeline)
	1. the process flow and pipline is to automately handle the problems in the release of software and  avoid the attack from haker.

	**CI/CD pipeline** automates software lifecycle and enhence the function of build and test.After packaged and released,it will merge the codes and test before delivery

	A **CI/CD pipeline** automates the software development lifecycle, ensuring faster and more reliable deployments. It begins with developers **writing code**, which is then **built** and **tested** automatically. Once validated, the software is **packaged and released**, followed by **deployment** to production. Finally, the system is **monitored** to track performance and resolve issues efficiently. 🚀

**"The DevOps pipeline follows a continuous feedback loop: Plan → Develop → Build → Test → Release → Deploy → Operate → Monitor. CI ensures frequent automated testing, while CD automates software deployment, either with (Continuous Delivery) or without (Continuous Deployment) manual approval.**
