# 个人博客实现规划

## 1. 信息架构

- 首页：Now 状态、三态内容入口、最近更新、知识地图、同步状态。
- 归档：按标题、摘要、主题、类型筛选。
- 文章页：强调中文长文阅读体验，侧栏说明同步来源。
- Now 页：公开最近关注事项。
- 知识地图：从路径、标签和主题聚合索引。
- 同步页：公开说明 Obsidian 到图床到博客的单向流程。

## 2. 视觉系统

- 字体：霞鹭文楷作为中文主字体。
- 色彩：纸色背景、墨色文字、苔绿主色、陶红提示色。
- 排版：左对齐、宽松行距、细分割线、少卡片、卡片圆角不超过 8px。
- 风格参照：Innei/Shiro 的三态内容、Fernweh 的中文静态博客气质、Now 页与数字花园的公开近况结构。

## 3. 同步策略

- 默认只同步 `publish: true` 的笔记。
- 支持 `--file` 指定单篇文档发布。
- 源库只读；生成文件写入站点目录。
- Obsidian wiki 图片和 Markdown 图片都会解析并上传到 provider。
- Obsidian 双链会改写成归档搜索链接。

## 4. 图床策略

- 本地预览：`local` provider 复制到 `public/media`。
- 生产图床：优先用已配置好的 PicGo，或用 Lsky Pro API。
- 不把 token 写入代码或文档，只通过环境变量读取。

## 5. 上线策略

- 阶段一：本地同步 + 静态构建。
- 阶段二：推送到 Git 仓库，Netlify/Vercel 自动构建。
- 阶段三：绑定域名、开启 RSS、补充评论/搜索/统计。
