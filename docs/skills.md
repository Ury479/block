# 可复用 Skills / Workflows

## 1. Obsidian 公开发布

目标：从本地 Obsidian 中生成可部署的公开博客副本。

步骤：

1. 在 `blog.config.mjs` 配置公开目录。
2. 运行 `npm run sync:publish-roots`。
3. 检查 manifest 的跳过原因。
4. 运行敏感扫描，不输出密钥值。
5. `npm run build`。
6. 提交并部署。

## 2. 付费型博客首页升级

目标：让公开内容自然承接咨询/陪跑服务。

步骤：

1. 首屏说明具体价值，不只写个人简介。
2. 展示公开资产规模和主题。
3. 放出 2-3 个具体服务，不写空泛 CTA。
4. 保留高质量免费内容入口。
5. 文章页减少营销干扰。

## 3. 内容安全发布检查

目标：避免把私钥、token、密码和登录配置发布到公网。

步骤：

1. 路径黑名单过滤。
2. 内容正则过滤。
3. 生成后再次扫描 `src/pages/posts`。
4. 只报告路径和数量。
5. 命中真实秘密时阻断发布。

## 4. Browser QA

目标：每次部署前用真实浏览器确认页面。

步骤：

1. 浏览器访问对标站点，记录首屏、导航、CTA 和内容结构。
2. 本地启动站点，截图桌面和移动端。
3. 检查关键页面状态码。
4. 检查横向溢出。
5. 部署后访问生产 URL。

## 5. 项目内 Skill

已抽取项目级 skill：`skills/obsidian-blog-publisher/SKILL.md`。

使用场景：

1. 批量同步 Obsidian 公开目录。
2. 新增付费服务、合集和内容类型。
3. 扩展 `/api/v1` 静态 JSON API。
4. 部署前做安全扫描、构建和浏览器验证。
